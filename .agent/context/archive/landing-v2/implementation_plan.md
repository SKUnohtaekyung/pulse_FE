# 전환 중심 랜딩페이지 인터랙션 UX 설계 및 실행안

본 계획은 제1차 대화의 요구사항(6대 완성도 원칙, 갭 분석, 전문가 워크스루 기반 검증, 에이전트 역할 분리)과 제2차 대화의 요구사항(CVR 방어, 엄격한 Hero 계층 관리, 통제된 Red Collar 인터랙션)을 완벽히 통합한 최종 설계안입니다. 

---

## 1. 수행 에이전트(Context)의 역할과 사용 시점
작업이 꼬이거나 권한이 중첩되지 않도록 프롬프트 체이닝에 참여할 6개 스킬의 구체적 롤을 명시합니다.

- **@/product-manager (PM)** 
  - **사용 시점**: 맨 처음 (분석 단계)
  - **목적 및 역할**: 현재 배포 코드 리딩 후 PRD(`about_pulse.md`)와 일치율 점검. 불일치 발생 시 1)구현 수정 2)기획 변경 3)추가 판단 필요 3가지로 분류. 디자인/개발 개입 금지.
- **@/UX Designer (UX 디자이너)**
  - **사용 시점**: PM 점검 이후 (설계 단계) 및 최종 단계 (검증 단계)
  - **목적 및 역할**: 전환/이탈 방어를 지향하는 정보구조 및 인터랙션 매핑. `animation.md`를 적극 차용하되 CVR을 훼손하는 요소는 필터링 적용. 완성 후 전문가 휴리스틱 및 시나리오 기반 워크스루 주도. 
- **@/UX Writer (UX 라이터)**
  - **사용 시점**: 레이아웃 골격이 나온 후 (콘텐츠 보완 단계)
  - **목적 및 역할**: 전환 중심의 직관적 마이크로카피 작성. 복잡성을 배제한 행동 유도 및 메시지 전달 보강.
- **@/Planner (기술 아키텍트)**
  - **사용 시점**: 설계 완료 직후 (전체 기술 파이프라인 수립)
  - **목적 및 역할**: GSAP/Lenis 최적화 및 리스크 방어 로직의 실제 코딩 태스크(JSON 형태) 도출. 기획 및 디자인 훼손 금지.
- **@/Frontend Developer (프론트엔드 개발자)**
  - **사용 시점**: 컴포넌트 단위 구현 돌입 시 (Phase 3)
  - **목적 및 역할**: 설계된 코드와 GSAP 인터랙션을 충돌 없이 짜맞추고, Hero 영역 UI 파손을 책임지고 방어함.
- **@/Performance Engineer (성능 최적화 엔지니어)**
  - **사용 시점**: 단계별 검증 및 프레임워크 렌더링 검토 시
  - **목적 및 역할**: 스크롤 Jank 회피 및 60fps 보장 디버깅 전담.

## 2. 작업 목표 및 6대 완성도 타겟
- **작업 목표 재정의**: 인터랙션은 “보조 수단”이며, 정보 전달과 전환(Conversion)이 가장 우선입니다.
- **6대 타겟 관점**: [정보 구조], [메시지 전달], [CTA 명확성], [신뢰 요소], [가독성], [인터랙터 일관성] 기준을 100% 충족해야 합니다.
- **인터랙션 허용 및 금지 기준**:
  - `animation.md` 및 Red Collar 레퍼런스를 차용하되, **CTA 주변부**, **핵심 가독 영역** 등 시선 분산 구역 내 과도한 모션 금지 (정보 지연 및 클릭 영역 이동 방지).
  - 오직 정보 이해를 돕고 인지 부하를 줄여주는 제한적 인터랙션(Fade, Hover Focus 등)만 허용.

## 3. 섹션별 전략 (레벨 정의 포함)

### Level 1: 전환 핵심 영역 (Hero 영역) → 인터랙션 최소화
- **적용 대상**: Hero Section (첫 화면 전체)
- **적용 인터랙션**: 페이지 진입 시 초단기 Fade-in, 폴리곤의 미세 Parallax Hover. 
- **안전 제한선 (Hero 고정)**: 
  - [Level A 절대 유지]: 기존 CTA의 위치 및 버튼 형태, 핵심 카피 의미.
  - [Level B 조건부 수정]: 가독성 확보를 위한 줄바꿈, 폰트 웨이트, 자간 조절, 시각적 계층 보강.
  - [Level C 추가 제약]: 랜딩 시 애니메이션/모션은 0.5초 안에 끝나야 하며 호버 틸트가 CTA 인접성을 해쳐선 안 됨.

### Level 2: 정보 전달 영역 → 보조 인터랙션만 허용
- **적용 대상**: Problem, Solution, 기능 안내 섹션 
- **적용 인터랙션**: 아코디언 드롭다운, Text Stagger, Hover Reveal (`animation.md` 기반).
- **안전 제한선**: 스크롤 하이재킹으로 사용자의 화면 제어권을 무력화하지 말 것. 애니메이션이 재생 중이어도 사용자가 원하면 즉시 스크롤 패스할 수 있어야 함.

### Level 3: 브랜딩/임팩트 영역 → 제한적으로 강한 인터랙션 허용
- **적용 대상**: Social Proof(신뢰/가상리뷰), 슬로건 트랜지션
- **적용 인터랙션**: Dark/Light 테마 반전, 3D 타이포그래피 등장 효과.
- **안전 제한선**: 스와이프나 클릭을 강제하지 않으며 단발성으로 강력하게 보여지고 사라지는 구조.

## 4. V2 아키텍처 전환 및 수익 모델 방어 전략 (Phase 1 갭 분석 반영)

### 4.1 V2 완전 단리 구조 (A/B Test Architecture)
현재 구동 중인 `LandingPage.jsx` 및 종속 컴포넌트 내부 코드는 절대 파괴하지 않습니다.
대신 `src/pages/LandingPageV2.jsx` 및 `src/components/landing_v2/` 디렉터리를 신설하여 100% 분리된 상태로 신규 코딩을 전개합니다.
- **라우팅 대안 제안**: 사용자는 "기존 랜딩 최하단 버튼"을 제안하셨으나, 중간에 이탈하는 고객은 테스트 화면을 아예 못 보게 되는 맹점이 있습니다. 따라서 **1) 기존 랜딩 하단에 큼직한 배너**를 두는 동시에, **2) 화면 우측 하단에 떠 있는 작고 세련된 플로팅 버튼(FAB) [PULSE 2.0 UX Preview]**을 달아서 스크롤 위치와 무관하게 언제든 V2로 바로 점프할 수 있게 듀얼 연결하는 것이 훨씬 효과적인 A/B 테스팅 방법입니다.

### 4.2 수익모델(Pro) 매칭 섹션 레이아웃 전략
과금 유도(Paywall)에서 오는 고객 저항을 없애기 위해, "더 내라"가 아니라 "더 넓은 무대를 지원한다"는 VIP 케어 느낌을 주어야 합니다.
- **전략**: 기본 3루프(Understand -> Create -> Action) 화이트 톤 섹션이 끝남과 동시에, 시각적 환기를 위해 배경을 **고급스러운 프리미엄 심층 다크 톤(Deep Blue / Black)**으로 반전(Theme Transition) 시킵니다.
- **카피 로직**: "입소문 나는 진짜 맛집들의 비결. 인플루언서 섭외, 직접 하지 마시고 PULSE Pro에 맡기세요."
- 이렇게 고급 파트너십 형태로 포장하여 Pro 요금제(리뷰 관리 및 협찬 연결)를 부드럽게 노출시킵니다.

## 5. 단계별 완료 기준 (Phase Criteria)
작업은 앞선 단계의 완료 기준이 충족되어야만 체이닝으로 넘어갑니다.

- **[Phase 1] 갭 분석 완료 기준**: 
  - 현재 베이스 코드와 기획서(`about_pulse.md`)의 불일치 여부를 도출하여 [구현 수정 / 기획 수정 / 논의]로 3분류 하였는가.
- **[Phase 2] UX 설계 완료 기준**: 
  - 6대 관점이 적용된 GSAP/인터랙션 시퀀스가 CTA 주변부 및 가독성 영역을 훼손하지 않음이 설계로 논증되었는가.
- **[Phase 3] 컴포넌트 프론트 구현 완료 기준**: 
  - 성능 프로파일링 결과 모바일 Scroll Jank 없이 60fps를 방어하는가. 
  - Level A/B/C 원칙을 준수했는가.
- **[Phase 4] 최종 UX 전문가 검증(QA) 완료 기준**: 
  - 6대 타겟 관점에 기반한 '시나리오 워크스루' 및 '휴리스틱 점검표'를 전체 통과(Passed) 했는가.

## 6. UX 검증 체크리스트 및 통과 방향 (Rule of Thumb)

- [ ] **1. CTA가 즉시 보이는가?** (통과 기준: 첫 로딩 스크롤 0px일 때 방해 없이 1초 내 Primary Button 선명하게 노출 / 실패 시 진입 애니메이션 즉시 단축)
- [ ] **2. 콘텐츠 이해까지 3초 이내 가능한가?** (통과 기준: 모션 대기 없이 곧바로 타이틀과 서브타이틀이 읽혀야 함 / 실패 시 노출 조기 Triggering)
- [ ] **3. 스크롤 피로도가 낮으며 강제 고정 없는가?** (통과 기준: 무거운 연산이나 스크롤 Locking 없음 / 실패 시 `Pin` 해제 로직 재검토)
- [ ] **4. 6대 관점 기준 인터랙션이 혼란을 유발하지 않는가?** (통과 기준: 마우스 움직임이 텍스트 리딩 시야를 방해하지 않음 / 실패 시 Z-index 및 스케일 조정)
