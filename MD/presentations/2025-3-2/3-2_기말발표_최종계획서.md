# 3-2 기말발표 최종계획서

## 1. 발표 목표

이번 발표의 목표는 아래 3가지를 명확하게 전달하는 것이다.

- PULSE가 어떤 문제를 해결하는 서비스인지 짧고 분명하게 설명한다.
- 지난 발표 이후 핵심 기능이 실제로 어디까지 구현되었는지 보여준다.
- 우리 팀이 AI를 활용한 협업형 개발 방식으로 빠르게 실험하고 고도화해 왔다는 점을 차별점으로 제시한다.

---

## 2. 발표 핵심 메시지

이번 발표에서 청중이 기억해야 할 핵심 메시지는 아래 세 가지다.

- `PULSE는 외식업 사장님의 마케팅을 자동화하는 플랫폼이다.`
- `핵심 기능은 이미 실제로 구현되고 있다.`
- `F12는 AI를 활용한 협업형 개발 방식으로 제품을 빠르게 고도화하고 있다.`

발표 전체를 한 문장으로 요약하면 다음과 같다.

`PULSE는 외식업 사장님의 마케팅 문제를 해결하는 자동화 플랫폼이며, 현재 핵심 기능을 구현해가고 있고, AI를 활용한 협업형 개발 방식으로 품질과 속도를 동시에 끌어올리고 있습니다.`

## 5-2. 2장 프로젝트 개요: 필요성 + 기획 + 핵심 기술

### 목적

가이드라인의 `프로젝트 개요` 항목을 한 장에서 충족한다.

### 포함 내용

#### 필요성

- 외식업 사장님은 마케팅이 필요하지만 지속적으로 실행하기 어렵다.
- 주요 장벽은 `지식`, `비용`, `실행`이다.

#### 기획

- PULSE는 분석과 생성, 실행을 하나의 흐름으로 연결하는 마케팅 자동화 플랫폼이다.
- 단순히 정보를 보여주는 것이 아니라 `다음 행동`까지 제안하는 구조를 가진다.

#### 핵심 기술

- 리뷰/상권 분석
- AI 기반 홍보 영상 생성
- 리뷰 답변 자동화
- React + Spring Boot + FastAPI + Kakao API + VEO3 기반 구조
- DSPy 및 AI 활용 방식은 5장에서 개발 방식과 함께 차별점으로 설명

### 발표 핵심 문장

`PULSE는 외식업 사장님의 지식·비용·실행 장벽을 줄이기 위해, 분석과 생성, 실행을 하나의 흐름으로 연결한 마케팅 자동화 플랫폼입니다.`

### 보조 문장

`저희는 리뷰와 상권 분석, AI 영상 생성, 리뷰 답변 자동화를 중심으로 React 프론트엔드와 Spring Boot, FastAPI 기반 구조를 설계했습니다.`

## 5-4. 4장 핵심 기능 연결 구조 + 시연영상

### 목적

기능들이 따로 존재하는 것이 아니라 하나의 루프로 연결된다는 점을 보여준다.

### 연결 구조

1. 손님 분석
2. 주변 상권 분석
3. 홍보영상 만들기
4. 리뷰 관리 답변
5. 인플루언서 매칭(Pro 확장)

### 설명 흐름

- 손님 분석으로 누구에게 팔지 정한다.
- 주변 상권 분석으로 우리 가게의 위치와 경쟁 맥락을 본다.
- 그 인사이트를 바탕으로 홍보 영상을 만든다.
- 유입 이후에는 리뷰 답변으로 고객 관계를 관리한다.
- 더 확장하면 인플루언서 매칭으로 외부 채널까지 연결한다.

### 발표 핵심 문장

`저희의 차별점은 기능 하나하나가 아니라, 분석 결과가 실행으로 이어지고 다시 다음 행동으로 연결되는 구조에 있습니다.`

### 시연영상 계획

시연은 별도 장이 아니라 이 장 안에 삽입한다.

#### 시연 구성

- 20초: 손님 분석 또는 상권 분석 화면
- 20초: 분석 결과 기반 영상 생성 흐름

#### 보조 시연 가능 항목

- 리뷰 관리 답변 화면
- 인플루언서 매칭 UI 화면

시연의 핵심은 기능 개수보다 `분석 -> 생성`의 연결 구조가 보이게 하는 것이다.

## 5-5. 5장 우리의 개발 방식

### 목적

이번 발표의 차별화 포인트인 `AI를 활용한 협업형 개발 방식`을 설명한다.

### 핵심 메시지

우리 팀은 AI를 사용자 기능에만 쓰는 것이 아니라, 개발 프로세스 자체에도 활용해 속도와 일관성을 높이고 있다.

### 포함 내용

- AI를 활용한 문서 정리와 기능 기획
- AI를 활용한 코드 탐색과 구현 초안 작성
- 반복 작업 자동화와 Git 작업 보조
- MCP와 skills를 통한 작업 문맥 연결과 품질 기준 정리
- DSPy 기반 접근은 프롬프트 구조 개선과 품질 향상 방향으로 검토 및 활용

### 발표 핵심 문장

`저희는 AI를 단순 생성 도구로만 쓰지 않고, 문서 정리, 기능 기획, 코드 탐색과 제작, 반복 작업 자동화, Git 작업 보조까지 개발 전반에 활용하고 있습니다.`

### 보조 문장

`즉, 개발을 사람의 감에만 맡기지 않고, 에이전트가 따라올 수 있는 문서와 절차를 함께 만들면서 팀의 개발 속도와 일관성을 높이고 있습니다.`

### 발표 시 유의점

- `AI agent`, `MCP`, `skills`라는 용어를 먼저 강조하지 않는다.
- 반드시 `속도`, `반복 실험`, `일관성`, `협업 효율` 같은 결과 중심 언어로 설명한다.

## 5-7. 7장 기대 효과 + 마무리

### 목적

서비스의 직접 효과와 확장 효과를 정리하고 발표를 마무리한다.

### 직접 효과

- 사장님의 마케팅 실행 부담 감소
- 콘텐츠 제작 시간과 진입장벽 완화
- 리뷰 응대 시간 단축 및 응답 일관성 향상
- 데이터 기반으로 다음 행동을 결정할 수 있는 운영 보조

### 확장 효과

- 외식업 소상공인의 디지털 마케팅 진입장벽 완화
- 지역 상권 단위의 디지털 전환 보조
- 장기적으로는 지역 상권의 디지털 격차 완화 가능성

### 발표 핵심 문장

`PULSE는 외식업 사장님의 마케팅 실행 부담을 줄이고, 데이터 기반 의사결정을 돕는 동시에, 지역 상권의 디지털 전환을 보조하는 방향으로 확장될 수 있습니다.`

### 마무리 문장

`PULSE는 외식업의 맥박을 읽고, 사장님의 성장 리듬을 만드는 서비스가 되고자 합니다.`
